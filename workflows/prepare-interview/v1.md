# Prepare Interview Workflow

Inputs: company name, optional stage (e.g. "technical screen"). All file access follows the storage-routing policy.

## Steps

1. Ensure base session context is loaded (applicant criteria, memory) — run `/context` if not.
2. Locate the application:
   - **OB1 active**: call `get_application("[company]")` — returns folder_prefix, profile, and linked file keys
   - **Fallback**: search `$APPLICANT_DIR/applications/` for a folder matching the company (case-insensitive, partial match OK)
   - If multiple matches exist, ask which one
3. Load the application's context files (per storage-routing):
   - `applications/[folder]/job-description.md`
   - `applications/[folder]/notes.md`
   - `profiles/[matched-profile].md` — positioning strategy
4. Produce the prep brief via skill `interview-prep` (it defines required reading, output structure, stage inference, no-fabrication rules, and the warm connection protocol)
5. End with: "Interview prep loaded for [Company] — [Stage]. What aspect do you want to work through?"

If the stage is not specified, the `interview-prep` skill infers the next upcoming stage from the Process section in notes.md.
