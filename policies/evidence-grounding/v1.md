# Evidence Grounding Policy

## Approved sources only

Source document content ONLY from:
- `profiles/[profile]/[profile]-CONTENT.md` — pre-compiled bullet library (all factual, pre-verified)
- `profiles/EXPERIENCE-REFERENCE.md` — canonical verified career history; source for Education and Certifications sections

`profiles/` is the working source of truth:
- `role-achievements.md` — **canonical source for all achievement bullet text and metrics** (verified or qualitative). Changes flow one-way: `role-achievements.md` → `EXPERIENCE-REFERENCE.md` → `*-CONTENT.md`. When achievement text is updated there, sync the corresponding bullet in EXPERIENCE-REFERENCE.md and any CONTENT files that use it.
- `[profile]/[profile].md` — positioning strategy and framing per profile
- `PROFILES-QUICK-REFERENCE.md` — fast-match index for screening; includes the `## Search Queries` table

`base-documents/` contains setup inputs (uploaded PDFs, interview notes, source resumes). It is **setup-only** — do NOT access it during normal workflow. Access it only when the applicant is adding new source material to support an updated or new profile.

## Two specific failure modes to avoid

- **Content library section headers are not job titles.** Headers like "AI Solution Architect - Presales Experience" in `-CONTENT.md` files are source material labels. Never render them as job entries.
- **Always verify role order** against the verified role list in `profiles/EXPERIENCE-REFERENCE.md` before generating.

## If a source is unreadable

If a PDF or source document is unreadable: ask the user for the information or an alternate format. Never reconstruct from memory.
