# Resume Format Rules

Formatting and structure rules applicable to any applicant. Applicant-specific content guidance lives in `$APPLICANT_DIR/base-documents/resume-content-guidance.md`.

---

## Voice & Tone

- Write in third person for bullets (no "I"); use active IC verbs: designed, implemented, architected, built, delivered, led, defined, drove, created
- Professional but direct — write like you talk, avoid corporate jargon
- Avoid management language when targeting IC roles: "led large teams," "departmental strategy," "oversaw transformation" → compress or omit
- Signal density: bullets follow **Action → Technical Domain → Context → Outcome** formula
- Embed keywords naturally — do not stuff
- Specific tools go in role bullets, not capabilities

---

## Section Structure

Every resume must follow this section order:

1. Name / contact block
2. Title line (role-specific)
3. Summary (3–4 sentences, tailored to role)
4. `## Core Capabilities` — categorized, see Capabilities Format below
5. `## RELEVANT EXPERIENCE` — never "Experience" or "Professional Experience"
6. `### Earlier Career` — subsection within Relevant Experience for roles ended 12+ years ago
7. `## Education`
8. `## Certifications`

Sections 7 and 8 must always be present. Content sourced from `$APPLICANT_DIR/profiles/EXPERIENCE-REFERENCE.md`.

---

## Role Ordering — Strict Reverse Chronological

- Most-recent-first, no exceptions
- Skipping roles is OK; out-of-order is not
- Roles that ended more than 12 years ago → group under `### Earlier Career`
- Earlier Career entries also follow reverse chronological order
- Always verify role order against `$APPLICANT_DIR/profiles/EXPERIENCE-REFERENCE.md`

---

## Detail Per Role

| Period | Bullets |
|---|---|
| Recent (last 10–12 years) | 5–7 bullets |
| Mid-career (12–20 years ago) | 2–4 bullets |
| Earlier Career (20+ years ago) | 1 bullet or title + dates only |

---

## Capabilities Format

Section header: `## Core Capabilities`

Format: bold category label followed by items separated by ` · ` — each category on its own paragraph line.

```
## Core Capabilities

**Category One:** Item · Item · Item · Item

**Category Two:** Item · Item · Item

**Category Three:** Item · Item · Item · Item · Item
```

Rules:
- 3–5 categories per resume
- 4–8 items per category
- No duplication across categories — if an item fits two, pick one
- Category labels should reflect the JD's language and the profile being targeted
- Use domain categories, not tool lists — specific tools go in role bullets
